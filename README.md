## 🎬 Design Inspiration

I analyzed the featured video, **[I Made a Custom GitHub Profile README Portfolio (and you can, too)](https://www.youtube.com/watch?v=9A8sQZDRn5o)** by **ForrestKnight**. The video presents the GitHub profile README as a lightweight portfolio: a strong identity block, badge-driven metadata, clean horizontal dividers, a dedicated tools section, project/media cards, and concise storytelling that makes a profile feel custom without leaving Markdown.

This README follows that style with:

| Video-inspired idea | Applied here |
| --- | --- |
| Portfolio-like hero | Name, role, location, and GitHub badges at the top |
| Visual scanability | Badges, tables, compact sections, and clear dividers |
| Developer identity | Practical focus on full-stack, AI, ML, and accessibility projects |
| Media as proof of taste | Featured YouTube thumbnail section |
| Story through projects | Repository cards built from local README/package scans and GitHub metadata |
